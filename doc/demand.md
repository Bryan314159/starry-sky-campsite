# 收藏夹可视化 — 需求文档

## 项目概述

将浏览器书签从一个枯燥的列表，变成一段从营地走向星空的沉浸式视觉旅程。产品形态为 Chrome 新标签页扩展，用户每次打开新标签页即进入该场景。

---

## 一、用户旅程

```
新标签页 → 场景一（营地·路牌）
              │
              │  点击某块路牌（书签文件夹）
              │  [后续迭代：小猫沿小路奔跑过场动画]
              ↓
           场景二（星空·书签）
              │
              │  点击星星 → 浏览器打开对应网址
              │  hover 星星 → 显示网站名称 + 微亮/放大效果
              │  点击小猫身后小路 → 返回场景一
              ↓
           回到场景一，可再次探索其他分类
```

---

## 二、数据来源与映射

### 数据源

浏览器原生书签（Bookmarks API），层级结构：

```
书签栏
├── 一级文件夹 A（分类）
│   ├── 网站 1（书签）
│   ├── 网站 2（书签）
│   └── ...
├── 一级文件夹 B（分类）
│   ├── 网站 3（书签）
│   └── ...
└── ...
```

### 映射关系

| 层级 | 书签结构 | 映射到 |
|------|---------|--------|
| 一级 | 书签分类文件夹 | 场景一的路牌指示牌 |
| 二级 | 文件夹内的具体书签 | 场景二的星星 |

- 仅读取一级文件夹和其下的直接书签。嵌套文件夹暂不处理，扁平化展示。

## 三、场景设计

### 场景一：营地 · 路牌

**定位**：用户进入新标签页的默认视图。

| 要素 | 说明 |
|------|------|
| 地面 | 草地/草坪，覆盖画面底部 |
| 核心元素 | 画面中央偏左，木质路牌柱，柱上钉着多块指示牌，各指向不同方向（经典路牌造型） |
| 空间 | 路牌前方一条小路向画面深处延伸，具有纵深感 |
| 氛围 | 温暖、白天或傍晚色调 |

**内容映射：** 每块指示牌显示一个一级书签文件夹的名称。指示牌指向方向各不相同，营造"多条路可选"的感觉。

**交互：**
- 点击某块指示牌 → 切换至场景二（后续加入过场动画）
- 搜索框融入场景中，输入关键词可触发浏览器默认搜索

**空状态：** 用户没有书签时，路牌上显示预先设计好的占位文字。

### 过场动画：沿路前行

> 注：本功能归入后续迭代，MVP 阶段不实现。

- 一只小猫角色出现在画面中
- 小猫从前景出发，沿着小路背对观众向前走
- 逐渐走向画面深处（透视缩小）
- 走到小路尽头 → 动画结束，切换至场景二

### 场景二：星空 · 书签

| 要素 | 说明 |
|------|------|
| 视角 | 小猫站在小路尽头的视角，抬头望向天空 |
| 天空 | 夜空，布满星星 |
| 氛围 | 宁静、开阔的星空夜景 |

**内容映射：**
- 每颗星星 = 所选分类下的一个具体书签（网站名 + 网址）
- 星星大小不一，营造自然的星空感
- **当前阶段：** 星星大小随机分布
- **后续阶段：** 结合数据采集后台，按书签使用频率动态调整大小和排布

**交互：**
- 点击星星 → 浏览器打开对应网址（在当前标签页跳转）
- Hover 星星 → 显示网站名称，伴随星星微亮/放大效果

### 返回机制

- 在场景二的地面区域，点击小猫身后的小路 → 切换回场景一（营地路牌页）
- 回到场景一后，用户可再次选择其他分类

## 四、视觉风格

### 总体方向

**手绘/插画风**。所有场景元素以手绘插画的方式呈现，色调温暖、笔触自然，像一幅会动的绘本插图。

- 非传统 UI，强调场景化交互
- 叙事感、沉浸感
- 场景一：温暖营地氛围
- 场景二：宁静星空氛围

### 角色

- 小猫（设计稿待提供）

## 五、技术方案

| 层面 | 选择 | 说明 |
|------|------|------|
| 产品形态 | Chrome 扩展，Manifest V3 | 优先支持 Chrome/Chromium |
| 渲染方案 | Three.js + React Three Fiber | 需真 3D 空间表现透视纵深和星空，R3F 做 React 化封装 |
| 前端框架 | React | 场景切换、书签数据管理等状态管理清晰 |
| 构建工具 | Vite | 构建快速，扩展开发插件成熟 |
| 样式呈现 | 手绘纹理 + toon material | 在 3D 空间中保持插画质感 |

### 关键约束

- 必须作为浏览器扩展运行
- 需要通过 Chrome Bookmarks API 读取书签
- 新标签页场景，要求加载快速、资源轻量
- 需保留搜索功能，搜索框融入场景画面

## 六、数据采集后台

> 注：本功能归入后续迭代，MVP 阶段不实现。

- 采集用户对每个书签（星星）的点击次数

- 非 MVP 范围，但 MVP 阶段需预留数据采集的接口设计
