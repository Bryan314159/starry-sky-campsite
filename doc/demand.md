# 收藏夹可视化 — 需求文档

## 一、项目概述

**星空营地** 是一个浏览器新标签页产品，它将浏览器的书签收藏从一个枯燥的列表，变成一段从温暖营地走向浩瀚星空的沉浸式视觉旅程。

用户每次打开新标签页，不是面对一个空白页面或快捷方式网格，而是"站"在一块手绘风格的营地中——眼前是木质路牌、柔软的草地、一条通向远方的小路。书签不再是一行行文字，而是路牌上的目的地、夜空中的星辰。每一次打开新标签页，都是一次小小的探索。

---

## 二、用户旅程

```
打开新标签页 → 场景一（营地 · 路牌）
                  │
                  │  看到木质路牌上钉着指示牌，每块牌子上写着一个书签文件夹的名字
                  │  指示牌各指不同方向，仿佛在说"选一条路吧"
                  │
                  │  点击某块指示牌
                  │  [后续迭代：小猫从画面边缘跑来，沿着小路向深处走去]
                  │
                  ↓
               场景二（星空 · 书签）
                  │
                  │  视角仿佛站在小路尽头仰望夜空
                  │  满天星星，每颗星对应所选文件夹里的一个书签
                  │  星星大小不一，疏密有致，像真实的星空
                  │
                  │  鼠标悬停某颗星 → 星星微微变亮、变大，旁边浮现网站名称
                  │  点击星星 → 在当前标签页打开对应网址
                  │  点击脚下小路 → 返回场景一
                  │
                  ↓
               回到营地，可以再次选择其他书签分类
```

## 三、数据映射

产品的数据来源是用户在浏览器中收藏的书签。书签的层级关系和产品场景的对应关系如下：

```
浏览器书签栏
├── 分类文件夹 A（一级，例如"开发工具"）
│   ├── 网站 1（书签）
│   ├── 网站 2（书签）
│   └── ...
├── 分类文件夹 B（一级，例如"设计灵感"）
│   ├── 网站 3（书签）
│   └── ...
└── ...
```

| 书签层级 | 映射到场景 | 表现形式 |
|---------|-----------|---------|
| 一级书签文件夹（分类） | 场景一：营地路牌 | 路牌柱上的指示牌，一块牌子对应一个文件夹 |
| 文件夹内的具体书签 | 场景二：星空 | 夜空中的星星，一颗星对应一个书签 |

**处理规则：**

- 只读取一级书签文件夹。嵌套在文件夹内的子文件夹不展示，保持扁平。
- 只读取文件夹内的直接书签。深层嵌套的书签不展示。
- 空文件夹（没有书签的文件夹）不展示。
- "其他书签"等系统默认文件夹不展示。

## 四、场景一：营地 · 路牌

### 4.1 画面描述

用户打开新标签页后首先看到的场景。

**地面与天空：**
画面底部是一片柔软的草地，覆盖整个视野的下半部分。草地有手绘笔触的质感，不是光滑的纯色块。天空是温暖的白昼或傍晚色调——浅橙、淡金、柔和的蓝，像日落前一小时的旷野。

**路牌柱：**
画面中央偏右位置，一根木质路牌柱从草地中矗立起来。柱子是原木质感，未经精细打磨，保留了树皮的粗糙纹理。高度约两层指示牌叠加的高度，给人"路标"的明确印象。

**指示牌：**
柱子上钉着多块木制指示牌，每块牌子的形状是经典的箭头形路牌（一头宽、一头尖，或矩形带斜角）。指示牌指向不同方向——有的偏左，有的偏右，有的微微上翘，有的略微下垂——营造出"多条路可选"的自然感觉。

每块指示牌上写着一个书签文件夹的名称（例如"开发工具""设计灵感""学习资料"）。文字是手写或手绘风格，与木板的质朴感统一。

**小路：**
路牌柱前方，一条泥土小路从画面近处向远方延伸，逐渐变窄，消失在地平线附近。小路有自然的弯曲，不是笔直的。这种纵深透视引导观众的视线从"脚下"走向"远方"。

**氛围关键词：** 温暖、安静、期待、出发前的那一刻。

### 4.2 画面布局示意

```
┌──────────────────────────────────────────┐
│                                          │
│       天空（暖色调傍晚）                    │
│                                          │
│                      ┌──────┐            │
│                      │ 路牌柱│            │
│                      │  ╱ ╲ │            │
│    小路 ──────────→  │ ╱   ╲│            │
│                      │╱     ╲│           │
│                      │ 指示牌 │           │
│                      │ 指示牌 │           │
│   ═══════════草地═════════════════════    │
└──────────────────────────────────────────┘
```

### 4.3 交互

| 操作 | 反馈 | 结果 |
|------|------|------|
| 鼠标悬停指示牌 | 指示牌微微上浮、轻微放大，木板的颜色稍微变亮 | 暗示可点击 |
| 点击指示牌 | 触发场景切换 | 进入场景二（星空），展示该文件夹下的书签 |
| 鼠标在其他区域 | 无特殊反馈 | — |

### 4.4 空状态

当用户没有任何书签文件夹时：

- 路牌柱依然存在，但上面没有指示牌。
- 取而代之的是 2-3 块预先设计的占位指示牌，上面写着引导性文字，例如：
  - "在这里添加你的第一个书签"
  - "在书签栏创建文件夹来整理你的收藏"
  - "然后回到这里，探索属于你的星空"
- 占位指示牌的视觉风格与正常指示牌一致，但颜色可以略淡或带有"等待被填写"的感觉。
- 小路、草地、天空等环境元素正常显示，不因空状态而缺失。

## 五、场景二：星空 · 书签

### 5.1 画面描述

用户点击某块指示牌后切换到的场景。

**视角：**
画面的视角相比场景一发生了明显变化。观众仿佛从营地出发、"走"到了小路深处，现在抬头望向天空。地面只占画面底部很小一部分，绝大部分画面是深邃的夜空。

**天空与星星：**
夜空是深蓝色到深紫色的渐变，上方最深（接近黑），地平线附近有一丝微光。天空中散布着几十到上百颗星星，每颗星都是温暖的金色或白色光点。

星星的分布不是均匀的——有的区域密集，有的区域稀疏，模仿真实夜空的自然错落感。每颗星的大小不同：有些是明亮的主星，有些是细小的光点。星星有微微的呼吸感（缓慢轻微的明暗变化），不是僵硬的固定光点。

**地面：**
画面底部保留一小片草地，以及小路延伸至此的末端。观众"站立"的位置就在小路尽头。小猫若在场景中，则坐或站在地面附近，背对观众或侧身，抬头望向星空。

**氛围关键词：** 宁静、辽阔、仰望、敬畏、发现。

### 5.2 星星与书签的对应

每颗星星代表所选书签文件夹中的一个书签：

- 星星的大小随机分布，自然的星空感。
- 后续迭代中，星星的大小和亮度可以根据书签的使用频率动态调整——常用的书签更亮更大，像夜空中最显眼的星辰。

### 5.3 交互

| 操作 | 反馈 | 结果 |
|------|------|------|
| 鼠标悬停星星 | 星星微微变亮、放大（约 1.3 倍），同时旁边浮现一个小标签，显示该网站的名称 | 预览书签内容 |
| 鼠标移出星星 | 星星恢复原始大小和亮度，标签消失 | — |
| 点击星星 | — | 在当前标签页跳转到该书签对应的网址 |
| 鼠标悬停小路区域 | 小路微微发亮，出现"沿路返回"的文字提示 | 暗示可点击返回 |
| 点击小路区域 | — | 切换回场景一（营地路牌） |

### 5.4 空状态

当选中的书签文件夹内没有任何书签时：

- 星空依然显示，但星星数量明显减少（或完全没有星星）。
- 画面中央显示一行提示文字，例如"这个文件夹里还没有书签"，引导用户返回营地选择其他分类。

## 六、过场动画

> **注：本节功能归入后续迭代，MVP 阶段不实现。MVP 阶段场景切换采用瞬时过渡。**

### 6.1 营地 → 星空（前进）

用户点击指示牌后：

1. 一只小猫角色从画面边缘走入场景，出现在小路起点附近。
2. 小猫背对观众，沿着小路向画面深处走去。
3. 随着小猫走远，它的身影在透视中逐渐变小。
4. 小路的纵深方向逐渐过渡为星空场景——天空从暖色调渐变为深蓝夜空，星星逐颗亮起。
5. 小猫走到小路尽头后停下，坐下或站立，抬头望向星空。
6. 动画完成，视角停留在场景二。

**动画时长参考：** 约 3-4 秒，不仓促也不拖沓。

### 6.2 星空 → 营地（返回）

用户点击小路区域后：

1. 动画反向播放：星空渐暗，暖色调天空渐亮。
2. 小猫转身，沿小路走回营地。
3. 画面回到场景一的完整营地视角。

### 6.3 小猫角色

- 小猫是产品的主角，是引导用户在两个场景之间穿梭的"向导"。
- 视觉风格：手绘感的小猫剪影或简单造型，与整体手绘场景统一。不需要逼真，需要可爱和温暖。
- 设计稿待提供。在 MVP 阶段先以简单形状占位。

## 七、视觉风格

> **画风参考基准：《罗小黑战记》。** 详细分析见 [画风参考文档](art-style.md)。

### 7.1 总体方向

**手绘插画风 / 治愈系。** 整个产品像一本会动的绘本插图——温暖的色调、手工笔触的纹理、质朴而不粗糙的造型。

核心体验不是"使用一个工具"，而是"走进一幅画"。

参考《罗小黑战记》的核心美学原则：

- **无影平涂** —— 物体和角色不使用光滑渐变阴影，采用 2-3 阶平涂色块。画面干净、有插画感。
- **前景简洁 + 环境精细** —— 路牌、星星、小猫等前景主体造型圆润简洁；天空、草地、小路等环境有丰富的手绘层次。
- **低饱和治愈色** —— 不刺眼、不荧光。色彩"温润克制"，像水彩画。
- **粗圆描边感** —— 物体边缘清晰可辨，像手绘线稿。

### 7.2 色彩方案

| 场景 | 主色调 | 辅助色 |
|------|--------|--------|
| 场景一（营地） | 暖草绿、木棕、奶油白 | 浅橙、淡金、柔蓝（天空） |
| 场景二（星空） | 深蓝紫、藏蓝 | 暖金（星光）、奶油白（提示文字） |

两个场景的色调形成明确的冷暖对比——营地的温暖 vs 星空的清冷。过场动画正是这座"冷暖之桥"。

### 7.3 质感指南

- **不要**光滑的塑料感、金属反光、渐变填充的"科技风"。
- **要**：
  - 纸张或画布的轻微纹理感
  - 铅笔或炭笔的不均匀笔触
  - 水彩或粉彩的柔和过渡
  - 木材的自然纹理
  - 草地的不规则斑点

### 7.4 文字风格

- 指示牌上的文字：手写体或手工感无衬线字体，仿佛用粉笔或油漆写在木板上。
- 界面提示文字：清晰可读的常规字体，但不能是冰冷的系统默认字体。
- 文字颜色与所在材质协调：木板上的字偏深棕，星空中的提示字偏暖金。
- 字体方向：无衬线但有手工感，不是机械的印刷体。像粉笔或白漆写在木板上，或像星光凝成的文字。

### 7.5 材质质感

| 元素 | 质感要求 |
|------|---------|
| 木头（路牌柱、指示牌） | 原木纹理，看得见年轮和色差，不光滑、不反光 |
| 草地 | 手绘笔触的不规则斑点，有深浅层次，不是均匀纯色 |
| 天空（白天） | 水彩晕染感的暖渐变，不是色块拼接 |
| 天空（夜空） | 深蓝到深紫的柔和过渡，不是纯黑底 |
| 星星 | 暖金色光点，带微微光晕，不是锐利的白色圆点 |
| 小路 | 泥土质感，有颗粒感和不规则边缘 |

## 八、交互设计（汇总）

### 8.1 场景切换

```
场景一 ──点击指示牌──→ 场景二
场景二 ──点击小路───→ 场景一
```

场景切换是产品最核心的交互，需要在"响应速度"和"过渡体验"之间平衡。MVP 阶段优先保证速度（瞬时切换），后续迭代加入过场动画。

### 8.2 搜索

产品中融入一个搜索框，形态上应是场景画面的一部分（而非一个突兀的 UI 控件）。

- **位置：** 画面底部居中，半透明，不遮挡场景主体。
- **外观：** 圆角、半透明背景、融入场景色调。在营地场景中偏暖，在星空场景中偏暗。
- **行为：** 用户在搜索框中输入关键词并按回车 → 触发浏览器的默认搜索引擎进行搜索。
- **可见性：** 两个场景中始终显示，不随场景切换而消失。

### 8.3 书签操作

- **打开书签：** 点击星星 → 在当前标签页跳转。不使用新窗口或新标签页，保持浏览的流畅感。
- **预览书签：** 悬停星星 → 浮出网站名称。不加载完整预览图（考虑性能和安全）。

### 8.4 无障碍

- 支持减少动画偏好：当用户在系统中开启了"减少动效"，所有动画应降级为瞬时过渡。
- 所有可交互元素应有明确的视觉提示（hover 态变化）。

## 十、边界情况与异常处理

| 情况 | 表现 |
|------|------|
| 用户没有任何书签 | 场景一显示占位指示牌，引导用户创建书签 |
| 用户有书签但没有文件夹 | 同上，引导创建文件夹 |
| 某个文件夹为空（无书签） | 该文件夹不展示在路牌上 |
| 某个书签的网址已失效 | 不做检测，点击后浏览器自行处理 |
| 用户快速连续点击多个指示牌 | 以最后一次点击为准，忽略中间状态的冲突 |
| 书签数量非常多（>100个） | 星星数量自动适配，避免过于拥挤 |
| 书签名称过长 | 指示牌上截断显示，超出部分用省略号 |
| 系统默认文件夹（"其他书签"等） | 不展示 |
