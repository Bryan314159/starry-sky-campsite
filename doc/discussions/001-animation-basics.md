## 问题一：动画是怎么实现的？我该怎么制作我想要的动画？

### 核心原理

所有动画的本质是**快速连续播放的静态画面**。浏览器每秒刷新屏幕约 60 次（60fps），每帧有细微变化 → 人眼感知为"动"。

技术流程：

```
requestAnimationFrame → 每帧调用你的函数 → 修改物体的属性 → Three.js 渲染 → 显示
```

### 在 React Three Fiber 中的实现

R3F 提供 `useFrame` hook，自动注册到渲染循环：

```jsx
import { useFrame } from '@react-three/fiber';

function MyStar() {
  const ref = useRef();

  useFrame((state, delta) => {
    // 每帧执行。delta = 距上一帧的时间（秒）
    ref.current.scale.x += (1.3 - ref.current.scale.x) * 5 * delta; // lerp 平滑过渡
  });

  return <mesh ref={ref}><sphereGeometry /><meshToonMaterial /></mesh>;
}
```

### 动画制作三步思考框架

1. **什么东西在变？** → 位置(position)、旋转(rotation)、缩放(scale)、颜色/亮度(material)、透明度(opacity)、相机(camera)
2. **怎么变？** → 瞬间切换、lerp 平滑过渡（最常用）、sin/cos 周期性运动（走路、浮动）、弹簧物理
3. **什么时候开始变？** → 点击、hover、挂载、另一动画结束

### 本项目动画分层

| 层次 | 内容 | 实现方式 | 优先级 |
|------|------|---------|:---:|
| 交互微动画 | 星星 hover 放大/发光、路牌晃动 | `useFrame` + lerp | MVP 必须 |
| 相机过渡 | 场景切换时相机飞行 | `useFrame` + lerp 或 GSAP | MVP 可选 |
| 小猫行走 | 过场动画：沿路走 + 透视缩小 + 四肢摆动 | 程序化摆动(sin/cos) 或 GLTF 骨骼动画 | 后续迭代 |

## 问题二：是否需要我主动添加一些图片？

### 不需要图片就能完成的（纯代码）

- 地面、小路、天空、路牌柱、指示牌、星星、小猫几何体拼接
- **所有动画逻辑和交互逻辑**：`useFrame`、点击检测、hover 效果、场景切换
- 结论：动画骨架能做到 100 分，**不依赖任何图片**

### 需要图片的地方

设计文档中定义的 6 张纹理，是画面"手绘质感"的核心来源：

| 纹理 | 尺寸 | 作用 | 没有时的效果 |
|------|------|------|-------------|
| `grass.png` | 512×512 | 草地手绘质感 | 纯绿色塑料感地面 |
| `wood.png` | 256×512 | 路牌柱木纹 | 棕色塑料管 |
| `sky-day.png` | 2048×1024 | 暖色调傍晚天空 | 纯色渐变天空 |
| `sky-night.png` | 2048×1024 | 深蓝夜空 | 纯深蓝色天空 |
| `signboard-blank.png` | 512×128 | 指示牌手绘木板 | 纯色方块 + 文字 |
| `star.png` | 64×64 | 星星光点质感 | 纯色发光小球（其实也不错） |

### 关键认知

> 图片 = 皮肤，动画 = 骨骼。骨骼不依赖皮肤工作。同样的 3D 代码，贴上手绘图就是"手绘绘本"，贴上纯色就是"开发者原型"。

## 问题三：纯代码方案能满足"画面精美、可按我想的设计"吗？

### 能力边界

纯色几何体 + `MeshToonMaterial` 的视觉上限是"色块风格"——干净的平涂风，但**无法做到手绘笔触的温暖质感**。

画面质感的构成比例：
```
最终画面 = 代码控制 50% + 纹理素材 50%

代码控制的部分：
  相机位置 → "观众站在哪"
  物体排列 → "路牌怎么布局"
  灯光颜色/强度 → "温暖傍晚还是清冷夜晚"
  动画参数 → "舒缓还是利落"

纹理控制的部分：
  手绘笔触、木纹、草地肌理、天空层次
```

### 实现路径对比

| 路径 | 方法 | 效果 | 推荐度 |
|------|------|------|:---:|
| 你画纹理 + 代码贴图 | Procreate/PS 手绘 → 贴到几何体上 | 完全掌控画风 | 推荐 |
| AI 生成纹理 | Midjourney/DALL·E 生成后调色统一 | 速度快，需会用 prompt | 可选 |
| 纯 Shader 程序化 | GLSL 着色器写纹理 | 算法感去不掉，学习曲线陡 | 不推荐 |
| 找设计师 | 外包纹理制作 | 效果最好，成本最高 | 可选 |

## 四、建议的开发策略

**代码和素材并行推进：**

1. 立即开始写所有代码 —— 场景搭建、动画系统、书签读取、交互逻辑
2. 临时用纯色 + 简易渐变色跑通全部功能
3. 同时准备 6 张纹理图（自己画 / AI 生成 / 找设计师）
4. 纹理到位后替换，只需改几行 `map: grassTexture` —— 画面直接质变
5. 然后精细调灯光、相机、动画参数到满意

## 五、后续讨论时的素材准备清单

当纹理素材准备好后，需要调整的关键参数：

- 相机位置和 FOV
- 灯光颜色（场景一暖黄 / 场景二蓝紫）
- 路牌指示牌的 Y 轴排布间距和旋转角度
- 星星的仰角范围和大小随机区间
- 动画的速度参数（lerp 系数）
