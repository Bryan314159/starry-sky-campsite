# 收藏夹可视化 — 规格文档 (Specification)

本文档以 GWT（Given-When-Then）格式定义所有功能场景的验收标准。每一条场景对应至少一条自动化测试。

---

## 一、场景一：营地 · 路牌

### SP1.1 正常加载 — 显示路牌

```
Scenario: 用户打开新标签页，有书签文件夹
  Given 浏览器中存在至少一个书签文件夹
  And 每个文件夹至少包含一个书签
  When 用户打开新标签页
  Then 画面显示营地场景
  And 画面包含：草地地面、暖色调天空、一条延伸向远方的小路
  And 画面中央偏右矗立一根木质路牌柱
  And 柱子上钉着 N 块指示牌（N = 有效书签文件夹数量）
  And 每块指示牌指向角度各不相同
  And 每块指示牌上显示对应文件夹的名称
  And 搜索框显示在画面底部居中位置
```

### SP1.2 悬停指示牌

```
Scenario: 用户悬停在指示牌上
  Given 用户在场景一
  And 指示牌正常显示
  When 用户将鼠标移到某块指示牌上
  Then 该指示牌微微上浮（沿 Y 轴位移）
  And 该指示牌轻微放大（约 1.02-1.05 倍）
  And 指示牌的亮度轻微提升
  And 鼠标样式变为 pointer（手型）
```

### SP1.3 移出指示牌

```
Scenario: 用户从指示牌上移开鼠标
  Given 用户正在悬停某块指示牌（已触发 hover 效果）
  When 用户将鼠标移出该指示牌
  Then 指示牌恢复原始位置、大小和亮度
  And hover 效果在 200ms 内平滑过渡
```

### SP1.4 点击指示牌 — 场景切换

```
Scenario: 用户点击指示牌进入星空场景
  Given 用户在场景一
  And 指示牌"开发工具"正常显示
  When 用户点击"开发工具"指示牌
  Then 画面切换至场景二（星空）
  And 切换为瞬时完成（MVP 阶段）
  And 夜空中显示"开发工具"文件夹下的所有书签星星
```

### SP1.5 搜索框可见

```
Scenario: 搜索框在场景一中始终可见
  Given 用户在场景一
  When 用户查看画面
  Then 画面底部居中显示搜索框
  And 搜索框为半透明样式，融入场景色调
  And 搜索框内显示占位提示文字（如"搜索..."）
```

### SP1.6 搜索触发

```
Scenario: 用户在搜索框中输入关键词并回车
  Given 用户在任意场景
  And 搜索框可见
  When 用户在搜索框中输入"GitHub"
  And 用户按下回车键
  Then 浏览器的默认搜索引擎在新标签页中搜索"GitHub"
  And 当前标签页保持不变（搜索不替换当前页面内容）
```

## 二、空状态

### SP2.1 无任何书签

```
Scenario: 用户没有任何书签
  Given 浏览器的书签栏为空（无文件夹、无书签）
  When 用户打开新标签页
  Then 画面显示营地场景（草地、天空、小路均正常渲染）
  And 路牌柱存在
  And 柱子上没有真实的指示牌
  And 柱子上显示 2-3 块占位指示牌
  And 占位指示牌颜色略淡于正常指示牌
  And 占位指示牌显示引导文字，如：
      - "在这里添加你的第一个书签"
      - "在书签栏创建文件夹来整理你的收藏"
      - "然后回到这里，探索属于你的星空"
```

### SP2.2 有书签但无文件夹

```
Scenario: 用户有散落书签但没有文件夹
  Given 浏览器书签栏中存在直接添加的书签（未归类到任何文件夹）
  And 不存在任何一级书签文件夹
  When 用户打开新标签页
  Then 按 SP2.1 空状态处理 ← 因为无文件夹 = 无指示牌可显示
```

### SP2.3 某个文件夹为空

```
Scenario: 书签文件夹存在但不含任何书签
  Given 浏览器中存在文件夹"待整理"
  And "待整理"文件夹内没有任何书签
  When 用户打开新标签页
  Then 该文件夹不出现在路牌指示牌上
  And 其他有书签的文件夹正常显示
```

### SP2.4 系统默认文件夹不展示

```
Scenario: 浏览器存在系统默认文件夹
  Given 浏览器中存在"其他书签"或"移动设备书签"文件夹
  When 用户打开新标签页
  Then 这些系统默认文件夹不出现在路牌指示牌上
```

## 三、场景二：星空 · 书签

### SP3.1 正常加载 — 显示星空

```
Scenario: 用户进入场景二，所选文件夹有书签
  Given 用户从场景一点击了"开发工具"指示牌
  And "开发工具"文件夹包含 5 个书签
  When 场景二加载完成
  Then 画面切换为星空夜景视角
  And 天空为深蓝色到深紫色的渐变
  And 画面中显示 5 颗星星（对应 5 个书签）
  And 星星分布在天空的上半部分（仰角 15° 以上）
  And 每颗星的大小不完全相同（随机分布，有自然的错落感）
  And 画面底部保留草地和小路区域
  And 搜索框仍可见
```

### SP3.2 星星大小分布

```
Scenario: 星星大小不一是自然随机感
  Given 场景二中显示多颗星星
  When 用户观察星空
  Then 星星大小在合理范围内随机变化
  And 大小分布接近自然星空（少量大星 + 多量小星）
  And 同一文件夹每次打开星星的位置和大小保持一致（基于书签 ID 的确定性随机）
```

### SP3.3 悬停星星

```
Scenario: 用户悬停在星星上
  Given 用户在场景二
  When 用户将鼠标移到某颗星星上
  Then 该星星放大至约 1.3 倍
  And 该星星亮度提升（发光更明显）
  And 星星旁边或上方浮现 Tooltip
  And Tooltip 显示该星星对应的网站名称
  And 鼠标样式变为 pointer
  And 以上效果在 200ms 内平滑过渡
```

### SP3.4 移出星星

```
Scenario: 用户从星星上移开鼠标
  Given 用户正在悬停某颗星星（已触发 hover 效果）
  When 用户将鼠标移出该星星
  Then 星星恢复原始大小和亮度
  And Tooltip 消失
  And 过渡在 200ms 内平滑完成
```

### SP3.5 点击星星 — 打开网址

```
Scenario: 用户点击星星打开书签网址
  Given 用户在场景二
  And 场景中显示"GitHub"书签对应的星星
  When 用户点击"GitHub"星星
  Then 当前标签页跳转到 https://github.com
  And 新标签页扩展页面被替换（正常浏览器行为）
```

### SP3.6 悬停小路 — 提示返回

```
Scenario: 用户悬停在小路区域
  Given 用户在场景二
  When 用户将鼠标移到画面底部的小路区域
  Then 小路区域微微发亮
  And 显示"沿路返回森林"或类似的返回提示文字
  And 鼠标样式变为 pointer
```

### SP3.7 点击小路 — 返回营地

```
Scenario: 用户点击小路返回场景一
  Given 用户在场景二
  When 用户点击画面底部的小路区域
  Then 画面切换回场景一（营地·路牌）
  And 切换为瞬时完成（MVP 阶段）
  And 路牌指示牌恢复到之前的状态（用户可重新选择文件夹）
  And 搜索框仍可见
```

### SP3.8 文件夹为空时的星星场景

```
Scenario: 进入星空场景但文件夹无书签
  Given 用户选中了一个没有书签的文件夹
  When 场景二加载
  Then 夜空正常显示
  And 画面中不显示星星（或显示极少量示意星）
  And 画面中央显示提示文字："这个文件夹里还没有书签"
  And 小路返回功能正常可用
```

## 四、场景切换

### SP4.1 前进切换（营地 → 星空）

```
Scenario: 完整的场景前进交互
  Given 用户在场景一
  And 路牌上显示 N 个指示牌
  When 用户点击其中一块指示牌
  Then 场景切换为场景二
  And 场景二展示所选文件夹对应的书签星空
  And 此前场景一中其他指示牌的状态不影响场景二
```

### SP4.2 返回切换（星空 → 营地）

```
Scenario: 完整的场景返回交互
  Given 用户在场景二
  When 用户点击小路区域
  Then 场景切换回场景一
  And 路牌恢复到场景一切换前的状态
  And 用户可再次点击不同的指示牌进入不同的星空
```

### SP4.3 多次切换不产生状态残留

```
Scenario: 用户反复进出不同文件夹
  Given 用户在场景一
  When 用户依次点击"开发工具"→ 返回 → "设计灵感"→ 返回 → "学习资料"
  Then 每次进入场景二都展示正确文件夹的星星
  And 每次返回场景一的路牌状态正确
  And 无闪烁、无画面残留、无内存泄漏
```

### SP4.4 快速连续点击不产生冲突

```
Scenario: 用户快速点击多块指示牌
  Given 用户在场景一
  When 用户在 100ms 内快速连续点击两块不同的指示牌
  Then 以最后一次点击为准
  And 场景二展示最后一次点击的文件夹的书签
  And 不出现中间状态闪烁或场景混乱
```

## 五、性能规格

### SP5.1 首帧渲染时间

```
Scenario: 新标签页首帧渲染性能
  Given 用户安装了该扩展
  When 用户打开新标签页
  Then 首帧画面在 500ms 内渲染完成
  And 用户可感知的加载时间不超过 1 秒
```

### SP5.2 动画帧率

```
Scenario: 场景中动画流畅度
  Given 用户在任意场景
  When 场景中存在 hover 过渡、星星呼吸等动画
  Then 动画帧率稳定在 60fps
  And 无明显掉帧或卡顿
```

### SP5.3 大量书签不影响性能

```
Scenario: 用户有大量书签（最多 200 个）
  Given 某个文件夹包含 200 个书签
  When 用户进入该文件夹的星空场景
  Then 场景正常渲染
  And 动画帧率不低于 30fps
  And 交互响应无延迟
```

## 六、画风一致性规格

### SP6.1 材质风格

```
Scenario: 所有 3D 元素符合画风要求
  Given 场景正常渲染
  When 检查场景中的 3D 元素
  Then 所有元素使用平涂/卡通材质（toon shading），无光滑渐变阴影
  And 色彩低饱和度、温润克制，无荧光色或纯原色
  And 场景一整体色调为暖色（草绿、木棕、奶油白、浅橙、淡金）
  And 场景二整体色调为冷色（深蓝、深紫、藏蓝、暖金星光）
  And 物体边缘清晰可辨，有描边/轮廓感
```

### SP6.2 文字风格

```
Scenario: 指示牌文字符合手绘风格
  Given 场景一中有指示牌显示文件夹名称
  When 用户查看指示牌上的文字
  Then 文字呈现手写/手绘感，不是机械印刷体
  And 文字与木板底色有适当的对比度（可读性）
  And 文字在木板上的定位合理，不偏离、不溢出
```

## 七、可访问性

### SP7.1 减少动画

```
Scenario: 用户启用了系统"减少动态效果"
  Given 用户在操作系统中开启了"减少动态效果"（prefers-reduced-motion: reduce）
  When 用户操作任何带动画的功能（hover、切换场景）
  Then 所有动画持续时间降至接近 0（≤ 1ms）
  And 功能本身不受影响（仍可点击、切换、打开书签）
```
